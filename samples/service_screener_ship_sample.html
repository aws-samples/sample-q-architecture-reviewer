🚨 Critical: Root 계정 MFA 미활성화

리소스 ID: User::root_id

현재 취약 상태: Root 사용자에 대한 Multi-Factor Authentication(MFA)이 비활성화되어 있습니다.

위험도 산정 근거: Root 계정은 모든 AWS 서비스와 리소스에 대한 완전한 액세스 권한을 가지므로, MFA 미설정 시 계정 탈취 위험이 극도로 높습니다.

비즈니스 영향도: 계정 전체 침해, 데이터 유출, 서비스 중단, 금전적 손실 가능성

해결 방안:
  1. AWS Console → IAM → Dashboard → "Activate MFA on your root account" 클릭
  2. Virtual MFA device 또는 Hardware MFA device 선택
  3. Google Authenticator 등 TOTP 앱 사용 권장
  4. MFA 설정 완료 후 Root 계정 사용 최소화
⚠️ IAM 사용자 액세스 키 90일 미순환

리소스 ID: User::sample_user_1, User::sample_user_2

현재 취약 상태: 액세스 키가 90일 이상 순환되지 않아 보안 위험이 증가했습니다.

위험도 산정 근거: 장기간 사용된 액세스 키는 노출 위험이 높으며, 정기적 순환이 보안 모범 사례입니다.

비즈니스 영향도: 액세스 키 노출 시 무단 API 호출, 리소스 조작, 데이터 접근 위험

해결 방안:
  1. AWS CLI: aws iam list-access-keys --user-name sample_user_1
  2. 새 액세스 키 생성: aws iam create-access-key --user-name sample_user_1
  3. 애플리케이션에서 새 키로 업데이트
  4. 기존 키 비활성화 후 삭제
  5. 90일 주기 자동 순환 정책 구현

리소스 ID: SG::sg-0123456789abcdef0, SG::sg-0abcdef123456789a

현재 취약 상태: 모든 포트(0-65535)가 0.0.0.0/0에 개방되어 있습니다.

위험도 산정 근거: 인터넷에서 모든 포트 접근 가능으로 무차별 공격에 노출

비즈니스 영향도: 서버 침해, 데이터 유출, DDoS 공격, 서비스 중단

해결 방안:
  1. 필요한 포트만 개방 (예: HTTP 80, HTTPS 443)
  2. 소스 IP 범위를 특정 CIDR로 제한
  3. 관리 포트(SSH 22, RDP 3389)는 VPN/Bastion 호스트를 통해서만 접근
  4. AWS CLI: aws ec2 revoke-security-group-ingress --group-id sg-0123456789abcdef0 --protocol all --port 0-65535 --cidr 0.0.0.0/0

리소스 ID: VPC::vpc-0123456789abcdef0, VPC::vpc-0abcdef123456789a

현재 취약 상태: VPC 네트워크 트래픽 로깅이 비활성화되어 있습니다.

위험도 산정 근거: 네트워크 활동 모니터링 불가로 보안 사고 탐지 및 분석 어려움

비즈니스 영향도: 보안 사고 대응 지연, 컴플라이언스 위반, 포렌식 분석 불가

해결 방안:
  1. VPC Flow Logs 활성화: aws ec2 create-flow-logs --resource-type VPC --resource-ids vpc-0123456789abcdef0 --traffic-type ALL --log-destination-type cloud-watch-logs --log-group-name VPCFlowLogs
  2. CloudWatch Logs 그룹 생성 및 보존 기간 설정
  3. S3 버킷으로 장기 보관 설정
  4. VPC Flow Logs 분석을 위한 CloudWatch Insights 쿼리 구성
